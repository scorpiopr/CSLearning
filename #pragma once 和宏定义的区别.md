#pragma once 宏定义都能用于防止头文件被重复包含
------------------------------
## 1. 为什么要使用它？（解决什么问题）
在复杂的 C++ 项目中，一个头文件可能会被多个不同的源文件或其他头文件包含（#include）。例如：

* Base.h 被 DerivedA.h 和 DerivedB.h 包含。
* main.cpp 同时包含了 DerivedA.h 和 DerivedB.h

如果没有保护机制，Base.h 的内容就会在 main.cpp 中被展开两次。这会导致编译器报错，提示 error: redefinition of 'class Base'（类重复定义）。
------------------------------
## 2. #pragma once 的工作原理
当编译器在编译某个源文件时，如果遇到了 #pragma once，它会记录当前头文件的物理路径。在接下来的编译过程中，如果再次遇到通过 #include 引入的同一个头文件，编译器就会自动跳过它，不再重复读取和解析。 [6] 
------------------------------
## 3. 与传统 #ifndef 的深度对比（面试高频）
在 #pragma once 出现之前，C/C++ 传统的方法是使用 宏定义保护（#ifndef / #define / #endif）。它们各有优缺点：
## 传统写法（宏保护）
```cpp
#ifndef MY_HEADER_H
#define MY_HEADER_H
// 头文件内容
#endif // MY_HEADER_H
```
## 两者的多维度对比

| 特性 | #pragma once | 传统 #ifndef 宏保护 |
|---|---|---|
| 标准支持 | 非 C++ 标准（由编译器提供扩展，但主流编译器都支持） | C/C++ 官方标准（跨平台与老旧编译器兼容性极佳） |
| 编译速度 | 更快（编译器通过物理路径直接跳过，不用每次都打开文件） | 稍慢（编译器必须每次都打开文件，并解析完宏判断） |
| 维护成本 | 极低（只需在文件开头加一行，不需要起名字） | 较高（每个文件需要独一无二的宏名，重名会导致文件失效） |
| 潜在风险 | 若同一文件在两处有多份物理拷贝，会被包含多次。 | 如果不同文件不小心用了相同的宏名，会导致其中一个文件被漏掉。 |
