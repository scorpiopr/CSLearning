##  64位系统代码示例
```cpp
class Base1 {
public:
    int val_1;
    virtual void func1() {}
};
class Base2 {
public:
    int val_2;
    virtual void func2() {}
};
// 1. 多重继承
class DerivedMultiple : public Base1, public Base2 {
public:
    int val_derived;
};
// ----------------------------------------------------
class Level1 {
public:
    int val_l1;
    virtual void funcL1() {}
};
class Level2 : public Level1 {
public:
    int val_l2;
    virtual void funcL2() {}
};
// 2. 单链式多层继承
class DerivedMultilevel : public Level2 {
public:
    int val_derived;
};
```
------------------------------
## 1. 多重继承的内存布局 (DerivedMultiple)
在多重继承中，子类对象内部会严格按照继承声明的顺序，将多个基类的子对象横向拼接在一起，最后再加上子类自身的成员变量。
## 📊 物理内存连续空间分布图 (总大小：32 字节)
```
偏移量 (Byte)   |  实际存储内容                          | 归属子对象
-------------------------------------------------------------------------
0x00 - 0x07    |  vptr_Base1 (指向 Base1 的虚函数表)    | [ Base1 子对象 ]
0x08 - 0x0B    |  int val_1  (4 字节)                   |
0x0C - 0x0F    |  [ 填充字节 Padding ] (4 字节，用于对齐) |
-------------------------------------------------------------------------
0x10 - 0x17    |  vptr_Base2 (指向 Base2 的虚函数表)    | [ Base2 子对象 ]
0x18 - 0x1B    |  int val_2  (4 字节)                   |
-------------------------------------------------------------------------
0x1C - 0x1F    |  int val_derived (4 字节)              | [ Derived 自身 ]
-------------------------------------------------------------------------
```
* 内存对齐（Padding）：因为后面紧跟着 Base2 的 vptr_Base2（8 字节，要求 8 字节对齐），所以前面的 val_1 后面必须被迫填充 4 字节的空白，造成了空间浪费。

## 2. 单链式多层继承的内存布局 (DerivedMultilevel)
在单链式多层继承中，由于是纵向单线代际传递，子类对象会垂直融合爷爷类和爸爸类的结构。它们共享同一个首地址的 vptr，而各层的成员变量则像套娃一样，按辈分从上往下紧凑排列。
## 📊 物理内存连续空间分布图 (总大小：24 字节)
```
偏移量 (Byte)   |  实际存储内容                          | 归属子对象
-------------------------------------------------------------------------
0x00 - 0x07    |  唯一的 vptr (融合了 L1, L2, 派生类的虚表) | 全链共享
0x08 - 0x0B    |  int val_l1 (爷爷辈的变量，4 字节)      | [ Level1 部分 ]
0x0C - 0x0F    |  int val_l2 (爸爸辈的变量，4 字节)      | [ Level2 部分 ]
0x10 - 0x13    |  int val_derived (自身变量，4 字节)     | [ Derived 部分 ]
0x14 - 0x17    |  [ 填充字节 Padding ] (4 字节，用于整体对齐)| 尾部补齐
-------------------------------------------------------------------------
```
* 极为紧凑的变量排列：看到 val_l1、val_l2 和 val_derived 了吗？它们是连续挨着排列的（0x08, 0x0C, 0x10）。因为它们都是 4 字节的 int，相互之间天然满足 4 字节对齐，编译器不需要在它们中间插入任何 Padding。直到最后，为了让整个对象的大小满足最大基本类型（vptr 8 字节）的整数倍原则，才在末尾统一补齐了 4 字节。

## 💡 大厂面试官的终极追问（高区分度）

追问：“在单链多层继承中，如果子类 DerivedMultilevel 重写了爷爷类的 funcL1()，这个虚函数表内部会发生什么？”


* 高分回答：
“虚函数表（Vtable）在内存中是一棵只读的指针数组。在单链继承中，虽然只有一个 vptr，但编译器会为最终的派生类生成一张完整的虚表。
如果子类没有重写，虚表对应的槽位（Slot）里存放的是 Level1::funcL1() 的函数入口地址；
一旦子类重写了该函数，编译器就会在编译期将该槽位里的地址强行覆盖（Overwrite）为 DerivedMultilevel::funcL1() 的地址。这就是 C++ 运行期动态绑定和多态的底层物理本质。”
