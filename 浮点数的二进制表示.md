# 一、 IEEE 754 浮点数核心公式
任何一个实数 $V$ 在计算机底层的二进制表示，都可以用以下公式推导出来：
$$V = (-1)^S \times M \times 2^E$$ 

* $S$ (Sign，符号位)：决定正负。$S=0$ 表示正数，$S=1$ 表示负数。
* $E$ (Exponent，指数位)：决定了浮点数的表示范围（即它是 2 的几次方），采用偏移码（Bias）方式存储。
* $M$ (Significand/Mantissa，尾数位/有效数字)：一个位于 $[1.0, 2.0)$ 之间的二进制小数，决定了浮点数的精度。

# 二、 单精度（float）与双精度（double）的内存布局
在内存中，这三个部分占用的二进制位（Bits）数是严格固定的：
## 1. 单精度 float（占用 4 字节 / 32 位）

  31     30       23 22                               0
+----+------------+-----------------------------------+

| S  |   E (8位)  |              M (23位)             |
+----+------------+-----------------------------------+


* 符号位 (S)：1 位
* 指数位 (E)：8 位（偏移量 Bias = 127）
* 尾数位 (M)：23 位

## 2. 双精度 double（占用 8 字节 / 64 位）

  63     62       52 51                               0
+----+------------+-----------------------------------+

| S  |  E (11位)  |              M (52位)             |
+----+------------+-----------------------------------+


* 符号位 (S)：1 位
* 指数位 (E)：11 位（偏移量 Bias = 1023）
* 尾数位 (M)：52 位

# 三、 底层存储的核心技术细节
## 1. 尾数的“隐藏首位”技术（Implicit Lead Bit）
因为规格化浮点数的有效数字 $M$ 的范围严格限定在 $[1.0, 2.0)$ 之间，所以它的二进制写出来必然是 1.xxxxxx 的形式（第一位必定是 1）。

* 优化机制：为了节省空间，IEEE 754 规定在内存中不存储这个开头的 1，只存储小数点后面的 xxxxxx。
* 效果：当 CPU 读取数据时，会自动在前面把这个 1. 补回来。因此，float 表面上尾数只有 23 位，但实际上拥有 24 位的精度。

## 2. 指数的“偏移量（Bias）”机制
指数 $E$ 可以是正数（如 $2^5$），也可以是负数（如 $2^{-5}$）。为了方便 CPU 硬件直接进行大小比较，IEEE 754 不用补码存负指数，而是引入一个固定的偏移量（Bias），强制将指数全部变成无符号正数。

* 内存里存的数字 = 实际指数 + 偏移量（Bias）
* float 的 Bias = 127。例如：实际指数是 2，内存里存 2 + 127 = 129（二进制 10000001）。

## 3. 十进制小数转换为二进制小数

采用的是“乘以 2 取整数”的规则。以十进制小数0.5为例。

------------------------------
### a. 为什么 0.5 的二进制是 0.1？
我们按照“乘 2 取整，顺序排列”的规则来推导：

* 第 1 步：拿小数部分 0.5 乘以 2：
$$0.5 \times 2 = 1.0$$ 
此时，计算结果的整数部分为 1，剩下的小数部分变成了 0。
* 第 2 步：因为剩下的小数部分已经为 0，计算到此结束。
* 第 3 步：把每一步得到的“整数部分”按顺序写在小数点后面，得到的就是 0.1。

------------------------------
### b. 深入二进制的数位权重（底层数学原理）
为了让你完全理解为什么它代表 0.5，我们可以对比一下十进制和二进制的数位权重：

* 十进制的小数位：
小数点后第一位的权重是 $10^{-1}$（即 $\frac{1}{10} = 0.1$）。
小数点后第二位的权重是 $10^{-2}$（即 $\frac{1}{100} = 0.01$）。
* 二进制的小数位：
小数点后第一位的权重是 $2^{-1}$（即 $\frac{1}{2} = 0.5$）。
小数点后第二位的权重是 $2^{-2}$（即 $\frac{1}{4} = 0.25$）。
小数点后第三位的权重是 $2^{-3}$（即 $\frac{1}{8} = 0.125$）。

因此，二进制小数 0.1 表达的数学意义就是：
$$1 \times 2^{-1} = 1 \times 0.5 = 0.5$$ 
同样地，如果是十进制的 0.75，它在二进制里就是 0.11，因为：
$$1 \times 2^{-1} + 1 \times 2^{-2} = 0.5 + 0.25 = 0.75$$ 

------------------------------
### c. 面试防坑：为什么说绝大多数十进制小数无法精确转换？
0.5 和 0.75 能够精确转换，是因为它们正好可以通过 $\frac{1}{2}$、$\frac{1}{4}$ 组合出来。但如果是十进制的 0.1 呢？我们来乘 2 试试：

* $0.1 \times 2 = 0.2$ $\rightarrow$ 取整 0
* $0.2 \times 2 = 0.4$ $\rightarrow$ 取整 0
* $0.4 \times 2 = 0.8$ $\rightarrow$ 取整 0
* $0.8 \times 2 = 1.6$ $\rightarrow$ 取整 1（剩下 0.6）
* $0.6 \times 2 = 1.2$ $\rightarrow$ 取整 1（剩下 0.2 $\rightarrow$ 开始循环！）

你会发现，十进制的 0.1 转换成二进制会变成 0.0001100110011... 无限循环小数！这就是为什么计算机在存储 0.1 时必须进行强制截断，进而导致浮点数天然具有精度误差的底层数学原因。

# 四、 实例演练：把十进制 5.5 转为 float 的二进制内存排布

   1. 确定符号：正数 $S = 0$。
   2. 转为二进制小数：
      * 整数部分： $5 = 101_{(2)}$
      * 小数部分乘以2取整数： $0.5 = 0.1_{(2)}$
      * 合起来： $5.5_{(10)} = 101.1_{(2)}$
   4. 写成二进制科学计数法（移动小数点）：
      * $101.1_{(2)} = 1.011_{(2)} \times 2^2$
      * 此时，实际指数为 2。
      * 尾数部分去除开头的 1.，剩下 011。
   4. 计算内存中的指数 E：
      * $E = 实际指数 + 127 = 2 + 127 = 129 \rightarrow$ 二进制为 10000001。
   6. 拼接 32 位内存数据：
      * S (1位) = 0
      * E (8位) = 10000001
      * M (23位) = 011 后面补 0 凑满 23 位 $\rightarrow$ 01100000000000000000000
      * 最终 32 位二进制：0100 0000 1011 0000 0000 0000 0000 0000
      * 转换为十六进制即为：0x40B00000。

# 五、 浮点数四大形态划分（高级考点）
根据指数位 $E$ 的二进制值，IEEE 754 浮点数在底层被严格划分为四种形态：

| 形态 | 指数位 $E$ 的值 | 尾数位 $M$ 的值 | 对应的数学含义 |
|---|---|---|---|
| 规格化数 | $0 < E < 255$ | 任意值 | 最普遍的常规浮点数，尾数带隐藏首位 1.。 |
| 非规格化数 | $E = 0$ | $M \neq 0$ | 极其接近 0 的数字，尾数带隐藏首位 0.（防止程序过早下溢）。 |
| 正负零 | $E = 0$ | $M = 0$ | 根据 $S$ 的不同，分为 +0.0 和 -0.0。 |
| 正负无穷大 | $E = 255$ | $M = 0$ | 计算溢出（如 1.0 / 0.0 得到 +inf）。 |
| NaN (非数) | $E = 255$ | $M \neq 0$ | 非法数学操作（如 0.0 / 0.0 或对负数开根号）。 |
