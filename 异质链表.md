传统链表只能存储同一种数据类型（如全是 int 或全是 ListNode*），而异质链表能够在一个链表中同时存储不同类型的数据（例如在一个链表里同时挂着整型、字符串、自定义类对象等）。
------------------------------
## 1. 方案一：面向对象多态法（最标准、最常用）

* 核心思路：
定义一个抽象基类作为链表节点的父类，让所有不同数据的子类都继承这个基类。链表节点中只保存基类指针（Base*）。利用 C++ 的运行时多态（虚函数）来调用不同子类的特定行为。
* 复杂度分析：
* 时间复杂度：插入/删除为 O(1)，访问/调用虚函数为 O(1)（需要经过一次虚函数表 Vtable 间接寻址）。
   * 空间复杂度：由于每个子类对象都有一个隐藏的虚函数表指针（vptr，64位系统下占 8 字节），会有额外的指针开销。

## 💻 现代 C++17 最佳实现：
```cpp
#include <iostream>
#include <vector>
#include <memory>
// 抽象基类
class BaseNode {
public:
    virtual ~BaseNode() = default;
    virtual void printInfo() const = 0; // 纯虚函数
};
// 子类 1：存储整型
class IntNode : public BaseNode {
    int data;
public:
    IntNode(int val) : data(val) {}
    void printInfo() const override { std::cout << "Int: " << data << std::endl; }
};
// 子类 2：存储字符串class StringNode : public BaseNode {
    std::string data;
public:
    StringNode(std::string val) : data(move(val)) {}
    void printInfo() const override { std::cout << "String: " << data << std::endl; }
};
int main() {
    // 现代 C++ 使用智能指针管理异质链表/数组，防止内存泄漏
    std::vector<std::unique_ptr<BaseNode>> heterogenous_list;
    
    heterogenous_list.push_back(std::make_unique<IntNode>(42));
    heterogenous_list.push_back(std::make_unique<StringNode>("Hello AI"));
    
    // 遍历异质链表
    for (const auto& node : heterogenous_list) {
        node->printInfo(); // 触发多态
    }
    return 0;
}
```

## 2. 方案二：现代 C++ 标准库法（C++17 std::variant）

* 核心思路：
如果不同类型的数量是有限且已知的，可以使用 C++17 引入的 std::variant（类型安全的共用体）。它可以在运行期安全地持有设定的几种类型之一，配合 std::visit 实现非继承关系的“多态”。
* 复杂度分析：
* 时间/空间复杂度：由于没有虚函数表指针（vptr），它避免了虚函数调用的开销，性能极高。但它的空间大小取决于最大类型成员的大小（类似于 union 的对齐和大小规则）。

## 💻 C++17 最佳实现：
```cpp
#include <iostream>
#include <vector>
#include <variant>
#include <string>
int main() {
    // 显式指定允许的类型范围
    using NodeData = std::variant<int, std::string, double>;
    std::vector<NodeData> heterogenous_list;

    heterogenous_list.push_back(100);
    heterogenous_list.push_back(std::string("C++17 Variant"));
    heterogenous_list.push_back(3.14);

    // 使用 std::visit 和 Lambda 表达式安全地访问数据
    for (const auto& data : heterogenous_list) {
        std::visit([](auto&& arg) {
            std::cout << "Data: " << arg << std::endl;
        }, data);
    }
    return 0;
}
```

## 3. 方案三：传统 C 语言法（void* 指针 + 类型标签）

* 核心思路：
在纯 C 语言或追求极致底层性能的场景下，由于没有类和多态，通常采用一个显式的结构体（struct）。内部包含一个显式的类型标记（enum）和一个万能指针 void*。
* 复杂度分析：
* 时间复杂度：由于需要显式通过 switch-case 判定类型，代码分支预测可能带来微小的性能波动。
   * 空间复杂度：由于没有虚函数表，每个节点只多占一个 enum 的大小（通常 4 字节），非常轻量。

## 💻 C 语言标准实现：
```cpp
#include <stdio.h>
#include <stdlib.h>
typedef enum { TYPE_INT, TYPE_FLOAT } DataType;
typedef struct HeteroNode {
    DataType type;
    void* data; // 万能指针，指向实际数据
    struct HeteroNode* next;
} HeteroNode;
void printList(HeteroNode* head) {
    HeteroNode* curr = head;
    while (curr != NULL) {
        // 面试核心：必须根据标签强制类型转换
        if (curr->type == TYPE_INT) {
            printf("C_Int: %d\n", *(int*)(curr->data));
        } else if (curr->type == TYPE_FLOAT) {
            printf("C_Float: %f\n", *(float*)(curr->data));
        }
        curr = curr->next;
    }
}
```

## 💡 异质链表大厂面试连环陷阱

   1. 内存泄漏（Memory Leak）陷阱：
   面试官一定会盯着方案一问：“为什么基类 BaseNode 的析构函数必须写成 virtual？”
   * 标准答案：如果不用 virtual，当通过基类指针删除子类对象时（如 delete base_ptr），只会调用基类的析构函数，而不会调用子类的析构函数，导致子类特有的内存数据发生严重泄漏。
   2. RTTI（运行时类型识别）性能消耗：
   面试官可能会问：“如果我拿到一个基类指针，想知道它到底是 IntNode 还是 StringNode 该怎么办？”
   * 答题思路：可以用 C++ 的 dynamic_cast<IntNode*>(ptr) 进行强转，如果成功说明是该类型。但要指出 dynamic_cast 会在运行时查虚函数表，有性能消耗。在工业级高频场景下，更推荐方案二的 std::variant 或手动维护类型标签。
