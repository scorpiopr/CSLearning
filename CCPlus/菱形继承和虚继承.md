## 什么是菱形继承？
假设我们有四个类，它们的继承关系像一个菱形：

* 类 A（祖先类）
* 类 B 继承了 A
* 类 C 继承了 A
* 类 D 同时继承了 B 和 C
```
      A (含有成员变量 int data)
     / \
    B   C
     \ /
      D
```
## 普通继承的致命问题
   1. 内存浪费：A 中的成员变量被复制了两份。
   2. 二义性（编译报错）：如果在 D 中直接访问 data，编译器会崩溃并报错：“对 data 的访问不明确”，因为它不知道你指的是从 B 继承过来的 data，还是从 C 继承过来的 data。

## 🛠️ 解决方案：虚基类
通过在继承时加上 virtual 关键字，将 A 声明为虚基类，就可以完美解决这个问题。

### 💻 代码示例

```cpp
#include <iostream>
using namespace std;

class D //基类
{
public:
    D(){cout<<"D()"<<endl;}
    ~D(){cout<<"~D()"<<endl;}
protected:
    int d;
};

class B:virtual public D // 虚继承
{
public:
    B(){cout<<"B()"<<endl;}
    ~B(){cout<<"~B()"<<endl;}
protected:
    int b;
};

class A:virtual public D // 虚继承
{
public:
    A(){cout<<"A()"<<endl;}
    ~A(){cout<<"~A()"<<endl;}
protected:
    int a;
};

class C:public B, public A
{
public:
    C(){cout<<"C()"<<endl;}
    ~C(){cout<<"~C()"<<endl;}
protected:
    int c;
};

int main()
{
    cout << "Hello World!" << endl;
    C c;
    c.d = 99;
    cout<<sizeof(c)<<endl;
    return 0;
}
```
### 📊 实际输出结果：
```
Hello World!
D()
B()
A()
C()
40
~C()
~A()
~B()
~D()
```
### 底层原理
* 普通继承：对象的内存布局是连续且静态的，子类包含父类的完整副本。
* 虚继承（Virtual Inheritance）：
  1. 编译器会为类 B 和类 A 隐式生成一个**虚基类表指针**（vbptr，Virtual Base Table Pointer），在内存中通常存放在程序的常量区（.rodata 节）。
  2. 这个指针指向一张**虚基类表**（vbtable）。表中记录了当前位置距离虚基类 D 的内存偏移量（Offset）。
     在内存中，vbtable 本质上是一个整型数组（通常为 int 或 ptrdiff_t 类型）。vbtable 内部结构通常包含以下固定项：
    ```      
      +---------------------------------------------+
      
      | [0] 项：当前 vbptr 距离当前子对象首地址的偏移 | -> 通常为 0 或负数
      +---------------------------------------------+
      
      | [1] 项：当前 vbptr 距离第一个虚基类的偏移量   | -> 正数（核心项）
      +---------------------------------------------+
      
      | [2] 项：当前 vbptr 距离第二个虚基类的偏移量   | -> 如果有多个虚基类
    ```
  3. 最终派生类 C 在初始化时，会根据这个偏移量，**把 D 的成员变量放在整块内存的最末尾**，且整个 C 对象中只有这一份 D。
  4. 无论通过 B 还是 A 去访问 D 的成员，都是通过指针加偏移量进行动态寻址。
* 构造顺序
**虚基类（Virtual Base Class）的构造函数必定在任何非虚基类（Non-virtual Base Class）之前调用**。
**虚基类的构造函数不再由直接子类（B 和 C）负责，而是必须由最终派生类（D）直接调用**。

  1. 首先：调用所有虚基类的构造函数（按照它们在继承列表中出现的先后顺序）。
  2. 其次：调用所有非虚基类的构造函数（按照它们在继承列表中出现的先后顺序）。
  3. 然后：调用当前类中所有成员变量的构造函数（按照它们在类中声明的先后顺序，与初始化列表中的顺序无关）。
  4. 最后：执行当前类自身构造函数的大括号 {} 内部的代码。

* 析构函数的执行顺序与构造函数严格完全相反

### 内存结构
当定义 C c; 时，c 对象内部的 B::vbptr 指向的 vbtable 结构如下：
1. 对象 c 的堆栈内存（实例数据）
```
地址 0:  [ 0x00403000 ]  <-- B::vbptr (存储了 vbtable 的内存地址)
地址 4:  [ int b ]  
地址 8:  [  A::vbptr  ]  
地址 12: [ int a ]  
地址 16: [ int d ]  <-- 虚基类 D 的数据区起点
```
2. 常量区中的 B_vbtable 内存结构（只读数据）
当 B::vbptr 指向地址 0x00403000 时，该只读内存块的内容为：
```
地址 0x00403000: [   0 ]  <-- 第 0 项：B::vbptr 距离 B 块首地址的偏移 (0 - 0 = 0)
地址 0x00403004: [  16 ]  <-- 第 1 项：B::vbptr 距离虚基类 D 的偏移 (16 - 0 = 16)
```
3. 常量区中的 A_vbtable 内存结构
同理，对象内部的 A::vbptr（位于地址 8）指向它专属的表：
```
地址 0x00403010: [   0 ]  <-- 第 0 项：A::vbptr 距离 A 块首地址的偏移 (8 - 8 = 0)
地址 0x00403014: [   8 ]  <-- 第 1 项：A::vbptr 距离虚基类 D 的偏移 (16 - 8 = 8)
```
4. 基类成员变量赋值底层原理
当你在 C++ 中写下 c.d = 99; 时，编译器在幕后将这行代码翻译成了基于 vbtable 结构的指针动态计算：
```cpp
// 1. 获取 B::vbptr 的地址
int* vptr = (int*)(&c); 
// 2. 解引用获取 vbtable 的首地址
int* vbtable = (int*)(*vptr); 
// 3. 取出 vbtable 中索引为 1 的项（即距离虚基类 D 的偏移量，此处为 16）
int offset = vbtable[1]; 
// 4. 将 obj 的首地址加上偏移量，转换为 D* 指针，并进行赋值
D* partD = (D*)((char*)&obj + offset);
partD->d = 99;
```
5. 为什么 sizeof(c) 是 40 字节？
在 64 位系统下，指针（虚基类表指针 vbptr 或类自身的 vptr）占据 8 字节，int 占据 4 字节。整个类 C 的物理内存排布遵循“8 字节对齐”原则，其内部组件拆解如下：

    1. B 的子对象区域（16 字节）：
      * B::vbptr（虚基类表指针）：8 字节。
      * int b：4 字节 $\rightarrow$ 额外填充 4 字节以对齐到 8 字节。
    2. A 的子对象区域（16 字节）：
      * A::vbptr（虚基类表指针）：8 字节。
      * int a：4 字节 $\rightarrow$ 额外填充 4 字节以对齐到 8 字节。
    3. C 自身的数据区域（8 字节）：
      * int c：4 字节 $\rightarrow$ 额外填充 4 字节以对齐到 8 字节。
    4. 共享的虚基类 D 的区域（8 字节）：
      * 虚继承将 D 剥离并挪到了内存最后方。
      * int d：4 字节 $\rightarrow$ 整个对象最末尾填充 4 字节满足结构体整体对齐。
  
💰 物理结构连加计算：
$$\text{Size} = 8 (\text{B的指针}) + 8 (\text{b与填充}) + 8 (\text{A的指针}) + 8 (\text{a与填充}) + 4 (\text{c}) + 4 (\text{d}) = 40 \text{ 字节}$$

## vptr (虚函数表指针) vs vbptr (虚基类表指针)

| 特性 | 虚函数表指针 (vptr) & 虚函数表 (vtable) | 虚基类表指针 (vbptr) & 虚基类表 (vbtable) |
|---|---|---|
| 核心目的 | 实现 运行时多态（动态绑定） | 解决 多重继承的菱形问题（消除多份拷贝） |
| 表中装的内容 | 函数指针（指向虚函数的地址） | 内存偏移量（整型数字） |
| 触发条件 | 类中含有 virtual 修饰的成员函数 | 类在继承时使用了 virtual 继承 |
| 寻址行为 | 找到表 -> 找到函数地址 -> 跳转执行 | 找到表 -> 找到偏移量 -> 计算内存物理地址 |
